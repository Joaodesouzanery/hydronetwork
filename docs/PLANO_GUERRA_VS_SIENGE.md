### CENARIO A — SER ADQUIRIDO (Acquisition Exit)

**Logica:** A Starian/Softplan ja gastou R$ 500 milhoes em aquisicoes. Eles COMPRAM empresas que preenchem lacunas no ecossistema. O saneamento/infraestrutura E uma lacuna.

**O que precisamos para ser um alvo atraente:**

| Requisito | Meta | Prazo |
|-----------|------|-------|
| Revenue (MRR) | R$ 50.000-100.000/mes | 18-24 meses |
| Clientes | 30-50 clientes pagantes | 18-24 meses |
| Nicho dominante | Lider em software de saneamento | 12-18 meses |
| Tecnologia diferenciada | Motor de calculo que eles nao tem | Ja existe |
| Retencao | Churn < 5% mensal | Continuo |
| Crescimento | 15-20% MoM nos primeiros meses | Continuo |

**Valuation estimado para aquisicao:**

```
Cenario conservador:
  ARR: R$ 1.200.000 (MRR R$ 100K)
  Multiplo SaaS Brasil: 5-8x ARR
  Valuation: R$ 6.000.000 - R$ 9.600.000

Cenario otimista:
  ARR: R$ 3.600.000 (MRR R$ 300K)
  Multiplo com premio estrategico: 8-12x ARR
  Valuation: R$ 28.800.000 - R$ 43.200.000

Cenario ideal (com traction + tecnologia unica):
  ARR: R$ 6.000.000+ (MRR R$ 500K+)
  Multiplo com premio: 10-15x ARR
  Valuation: R$ 60.000.000 - R$ 90.000.000
```

**Como chamar a atencao deles:**
1. Dominar o nicho de saneamento publicamente (LinkedIn, eventos ABES, FENASAN)
2. Conquistar clientes que o Sienge tambem atende (construtoras de infra)
3. Ser mencionado em comparativos "Sienge vs ConstruData"
4. Participar do ConstroSummit (evento do Sienge) como palestrante/expositor
5. Publicar conteudo que o Sienge nao cobre (normas hidraulicas, EPANET, LPS)

**Riscos deste cenario:**
- Eles podem simplesmente ignorar e construir internamente
- Valuation pode ser menor que o esperado
- Perda de autonomia pos-aquisicao

### CENARIO C — DERROTAR E SUBSTITUIR (Market Disruption)

**Logica:** Seguir o playbook classico de disrupção de Clayton Christensen. Comecar por baixo, num nicho que o incumbente ignora, e subir.

```
TEORIA DA DISRUPCAO APLICADA:

              MERCADO
              ▲
              │
    ALTO      │         ┌─────────────┐
    (Grandes  │         │   SIENGE    │
    construct.)│        │ (incumbente)│
              │         └──────┬──────┘
              │                │ Foco aqui
              │                │
              │                │
    MEDIO     │    ┌───────────┤
    (Medias)  │    │           │
              │    │    ┌──────▼──────────────┐
              │    │    │  CONSTRUDATA sobe    │
              │    │    │  para ca (2028+)     │
              │    │    └─────────────────────┘
              │    │
    BAIXO     │    │
    (Pequenas │    ▼
    consult.) │ ┌──────────────────────┐
              │ │  CONSTRUDATA comeca  │
              │ │  aqui (2026-2027)    │
              │ └──────────────────────┘
              │
              └──────────────────────────────► TEMPO
```

**Fases da Disrupcao:**

**Fase 1 — Ignorado (2026):** O Sienge nem sabe que existimos. Perfeito. Conquiste o nicho de saneamento/consultoria pequena enquanto eles nao estao olhando.

**Fase 2 — Ridicularizado (2027):** "Ah, aquele software de saneamento? Nem e ERP de verdade." Otimo. Continue crescendo.

**Fase 3 — Combatido (2028-2029):** Eles notam que estamos roubando clientes de infraestrutura. Tentam reagir. Mas ja e tarde — nosso motor de calculo tem 3 anos de vantagem.

**Fase 4 — Aceito (2030+):** Somos a referencia em engenharia de infraestrutura. Eles sao a referencia em incorporacao. O mercado se divide.

As concessionarias (CAESB, SABESP, SANEPAR, COPASA, BRK, Igua, Aegea) sao o maior premio. Uma unica concessionaria pode valer R$ 150.000-500.000/ano.

| Passo | Acao | Timeline |
|-------|------|----------|
| 1 | Conquistar construtoras que TRABALHAM para concessionarias | 2027-2028 |
| 2 | As construtoras pedem para a concessionaria aceitar dados da ConstruData | 2028 |
| 3 | Concessionaria conhece a plataforma via construtora | 2028-2029 |
| 4 | Oferecer modulo de FISCALIZACAO para a concessionaria | 2029 |
| 5 | Concessionaria contrata para padronizar dados de todas as construtoras | 2029-2030 |
| 6 | Lock-in: todas as construtoras da regiao precisam usar ConstruData | 2030 |

**Modulo de conformidade como cavalo de Troia:**
O modulo CAESB/SABESP ja existe. Quando uma construtora entrega dados via ConstruData para a concessionaria, a concessionaria percebe a qualidade e padronizacao. Isso cria demanda de baixo para cima (bottom-up adoption).

#### 3.4 Funding — Quando e Se Levantar Capital

**Decisao critica:** Levantar investimento OU crescer com receita propria (bootstrapped)?

| Cenario | Bootstrapped | Com investimento |
|---------|-------------|-----------------|
| **Velocidade** | Lenta mas sustentavel | Rapida mas com pressao |
| **Controle** | 100% do fundador | Diluicao 20-40% |
| **Risco** | Menor (sem divida) | Maior (expectativas de crescimento) |
| **Meta de exit** | Flexivel | 5-7 anos (expectativa do investidor) |
| **Ideal para** | Cenario B (crescer ate igualar) | Cenario C (derrotar) |

**Se decidir levantar:**
- Seed: R$ 2-5 milhoes (com traction de R$ 100K MRR)
- Serie A: R$ 10-30 milhoes (com R$ 500K MRR)
- Investidores ideais: Canary, Kaszek, Valor Capital, Astella
- Tese: "Sistema operacional da infraestrutura de saneamento do Brasil"

**Se decidir bootstrapped:**
- Reinvestir 100% da receita em produto + marketing
- Manter custo fixo minimo (time enxuto, sem escritorio)
- Crescer organicamente, sem pressao de investidor
- Vantagem: nao dilui, nao perde controle, nao tem board

**Recomendacao:** Bootstrap ate R$ 200K MRR. Depois decidir com dados reais.
